# Security Policy

Thank you for helping improve the security of OrchiCyb AI ATTACK.

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability affecting OrchiCyb AI ATTACK, please report it responsibly.

Please include as much information as possible, such as:

- Description of the issue
- Steps to reproduce
- Potential impact
- Screenshots (if applicable)
- Suggested remediation (optional)

Please do **not** disclose vulnerabilities publicly before they have been reviewed.

---

## Contact

Security reports can be submitted via:

- GitHub Security Advisories (if enabled)
- GitHub Issues (for non-sensitive security-related questions)

For sensitive reports, please contact:

**Email:** dominika.orchicyb@gmail.com 

---

## Scope

This repository primarily serves as a project showcase and documentation repository.

It contains:

- project documentation
- screenshots
- architecture concepts
- feature descriptions
- links to the application

The application source code is **not** included in this repository.

As a result, vulnerability reports related to the contents of this repository are generally limited to:

- exposed sensitive information
- insecure documentation
- accidental credential disclosure
- broken security-related links
- vulnerabilities affecting the publicly available application

---

## Responsible Disclosure

Please allow reasonable time to investigate reported issues before any public disclosure.

Valid reports will be acknowledged and addressed as appropriate.

---

## Supported Versions

The latest publicly available version of OrchiCyb AI ATTACK is supported.

Earlier demonstration builds may no longer receive updates.
